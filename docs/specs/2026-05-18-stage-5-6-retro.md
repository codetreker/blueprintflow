## What's still deferred (post-v1)

Carried from the Stage 5+6 plan's "Out of Scope" section, plus
what Stage 5+6 surfaced and we punted on:

1. **14 of 21 v6 skills' full content migration into Pack protocols** —
   bulk migration is post-v1; the demo named the next 2–3 candidates.
2. **LLM-driven NL transcription** for the standalone CLI — works
   inside the SKILL.md orchestrator only; CLI requires verb-first.
   v1.1.
3. **Sibling-Pack npm discovery** — `pack-discovery` scans repo-local
   only. v1.1+ when a real second Pack exists.
4. **Cron-driven verbs** (`sweep`, `intake`) — explicitly post-v1.
5. **Real `npm publish`** — first publish is owner-driven, post-rc.1
   sign-off.
6. **Child-run dispatch for `loop`** — `loop` verb still returns
   `{deferred: true}`; needs a "node spawns child runs" primitive
   that's larger than v1 should swallow.
7. **Per-WO live-state file** — Stage 4 retro item #3.
8. **Structured edge payloads `{verdict, scope}`** — Stage 4 retro
   item #4.
9. **Multi-line YAML in `wo.md`** — Stage 4 retro item #7; needs a
   YAML library swap. v1.0.1.
10. **`bf resume --last` ergonomics** — Stage 4 retro item #6. v1.0.1.
11. **Demo-pollutes-runner-repo** (Stage 5 finding #6) — future demo
    runs should target a scratch repo.

## How to actually try it

```bash
git clone <this repo>
cd blueprintflow
# (optional) npm install -g . for a global `bf` symlink

node bin/bf.mjs create "your first task" --pack product-engineering --schema task
node bin/bf.mjs execute <wo-id>

# In Claude Code with this repo open:
# /bf <wo-id>        — runs the orchestrator with real subagents
```
