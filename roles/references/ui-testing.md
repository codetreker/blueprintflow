# UI Testing Reference

Use this reference when a tester reviews UI behavior. It guides judgment and evidence expectations for the reviewed UI scope; it does not require every item here to block every task.

## Focus Areas

- UI journeys: verify the expected entry points, primary task path, alternate paths, cancellation, completion, and recovery behavior.
- states: check loading, empty, populated, disabled, validation, error, success, and permission-related states that are relevant to the task.
- input behavior: exercise form controls, keyboard actions, validation timing, destructive-action safeguards, and preserved or cleared values.
- accessibility: review labels, focus order, keyboard reachability, status feedback, contrast-sensitive states, and assistive-technology cues where applicable.
- responsive behavior: inspect the UI across the supported viewport and content-size range named or implied by the task.
- layout reasonableness: confirm spacing, alignment, density, truncation, wrapping, and content hierarchy make the UI understandable without overlapping or unstable controls.
- UX validation: check whether the interaction gives users enough context, feedback, and recovery options to complete the intended task without hidden assumptions.
- state consistency: verify UI state matches underlying data, navigation, submitted values, pending work, errors, and refreshed results.
- evidence quality: prefer focused screenshots, recordings, automated assertions, or manual check notes that show the reviewed states and the reason each accepted criterion is satisfied.
